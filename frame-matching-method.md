# 样片配框方法

---

## 第一步：像素采样 + 主色提取

将 N 张样片的像素合并为一个像素池，统一进行一次 **k-means 聚类**（k=5）。

### 采样模式（三选一）

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| `full` | 取全图所有像素 | 样片整体色调均匀 |
| `edge` | 只取四条边的像素带（边缘占比 10~20%） | 样片放在画框内，边缘与画框直接相邻 |
| `center` | 只取中心矩形区域 | 关注主体色调，忽略边缘背景 |

### 什么是 k-means

k-means 是一种将大量数据点自动归为 k 组的算法。用在颜色提取上，就是把数百万个像素的颜色，自动归纳为 5 个"最有代表性的颜色"（聚类中心），同时统计每种颜色占总像素的比例（权重）。

多张样片的像素先合并再做一次 k-means，确保提取的是所有样片的整体色调，而不是每张单独计算后再拼凑。

### 输出

- **样片主色**：5 个颜色，每个附带占比权重
- **样片明度**：`bright` / `dark`

### 明度判断：WCAG 加权亮度

WCAG（Web Content Accessibility Guidelines，网页无障碍指南）定义了一套感知亮度公式，对 RGB 三通道先做 gamma 线性化，再按人眼感知权重求和：

```
L = 0.2126 × R_linear + 0.7152 × G_linear + 0.0722 × B_linear
```

人眼对绿色最敏感（权重 0.715），对蓝色最不敏感（权重 0.072）。
对 5 个主色按占比加权平均，得到整体感知亮度，与阈值比较得出 `bright` / `dark`。

## 第二步：彩度过滤，确定代表色

对 5 个样片主色，每个计算 **OKLCH 彩度 C**。

### 什么是 OKLCH

OKLCH 是一种感知均匀的颜色空间，由三个分量构成：

| 分量 | 含义 | 范围 |
|------|------|------|
| L | 亮度（Lightness） | 0~1 |
| C | 彩度（Chroma） | 0~0.4 |
| H | 色相（Hue） | 0°~360° |

**"感知均匀"** 意味着数值上相同的差距，人眼看来视觉差异也一致——无论是红色区域还是蓝色区域，30° 的色相差距感知相同。

相比 HSV 的 S（饱和度），OKLCH 的 C 更准确反映颜色的鲜艳程度，不受亮度干扰。

### 过滤逻辑

取彩度 C 最大的颜色作为**样片代表色**，用其色相 H 参与后续匹配。

```
代表色 = max(主色1~5, key=OKLCH_C)
```

### 特殊情况：全部低彩度（is_desaturated = True）

若所有主色的 OKLCH C 均低于阈值（默认 0.06）：

- `is_desaturated = True`
- **跳过色相匹配，使用兜底方案**

**原因：** 当所有颜色都接近灰 / 黑 / 白时，色相 H 在数学上虽然有值，但实际是噪声——一个几乎是灰色的像素，其 H 值可能因极微小的 RGB 差异而跳动到任意角度，没有实际意义。强行用这个 H 匹配画框只会引入随机误差。

**兜底方案：** 只按明度匹配——`bright` 样片配 `dark` 画框，`dark` 样片配 `bright` 画框，一深一浅形成视觉对比。

---

## 第三步：配框匹配

画框库中每个画框预先维护两个属性：

| 属性 | 类型 | 说明 |
|------|------|------|
| 画框代表色 | HEX | 画框的主色，人工标注或离线计算 |
| 画框明度 | `bright` / `dark` | 画框整体深浅 |

### 正常流程（is_desaturated = False）

将样片代表色与每个候选画框的代表色，在 **OKLCH 色相空间**中计算环形角度距离：

```
distance = min(|H_样片 - H_画框|, 360° - |H_样片 - H_画框|)
```

环形计算确保跨 0°/360° 边界的颜色距离正确，例如 10° 与 350° 距离为 20° 而非 340°。

按距离从小到大排序，输出最优画框序号。

### 兜底流程（is_desaturated = True）

只看明度：

| 样片明度 | 匹配画框明度 |
|---------|------------|
| `bright` | `dark` |
| `dark` | `bright` |

---

## 流程总览

```
N 张样片
    │
    ▼
像素采样（full / edge / center）
合并像素池 → k-means（k=5）
    │
    ├─→ 样片主色（5个）
    │       │
    │       ▼
    │   OKLCH 彩度计算
    │       │
    │       ├─ 全部低彩度 → is_desaturated=True
    │       │                       │
    │       │               兜底：明度对比匹配
    │       │
    │       └─ 正常 → 取最高彩度颜色为代表色
    │                       │
    │               OKLCH 色相距离排序
    │                       │
    │               输出最优画框序号
    │
    └─→ 样片明度（WCAG）→ bright / dark
                │
        兜底流程时使用
```
